# 経理・財務AIエージェント

freee APIから会計データを取得し、経営判断に使えるレポートを自動生成するCLIツールです。

## 目的

1. freee APIから会計データを自動取得
2. 財務状況を構造化・可視化
3. 月次会計レポート・財務レポートを自動生成
4. 経営判断に使えるコメントを出力
5. 将来的に経営者との対話型インターフェースに拡張可能

## 主要ユースケース

- 月次の経営状況把握（売上・利益・現預金）
- 前月比較による異常検知
- 銀行評価の簡易判定
- 経営者向けの財務コンサルティングコメント生成

## セットアップ

```bash
# 依存パッケージのインストール
npm install

# 環境変数の設定
cp .env.example .env
# .env を編集して freee API の認証情報を設定
```

## 環境変数

| 変数名 | 説明 | 必須 |
|--------|------|------|
| `FREEE_CLIENT_ID` | freee アプリのClient ID | ○ |
| `FREEE_CLIENT_SECRET` | freee アプリのClient Secret | ○ |
| `FREEE_REDIRECT_URI` | OAuth コールバックURL | ○ |
| `FREEE_ACCESS_TOKEN` | アクセストークン | ○ |
| `FREEE_REFRESH_TOKEN` | リフレッシュトークン | ○ |
| `FREEE_COMPANY_ID` | 事業所ID（省略時は一覧から選択） | - |
| `AI_PROVIDER` | コメント生成方式（`template` / `anthropic`） | - |
| `ANTHROPIC_API_KEY` | Anthropic API Key（将来拡張用） | - |
| `OUTPUT_DIR` | レポート出力先（デフォルト: `./output`） | - |
| `OUTPUT_FORMAT` | 出力形式（`json` / `markdown` / `both`） | - |

## freee API 接続方法

1. [freee Developers](https://developer.freee.co.jp/) でアプリを作成
2. Client ID / Client Secret を `.env` に設定
3. 認証URLにアクセスして認可コードを取得
4. アクセストークン・リフレッシュトークンを `.env` に設定

認証URLは `npm run dev -- --mock` なしで実行すると表示されます。

## 実行コマンド

```bash
# モックデータでレポート生成（API接続不要）
npm run dev -- --mock --month=2026-02

# freee API 接続でレポート生成
npm run dev -- --companyId=12345 --month=2026-02

# 出力先を指定
npm run dev -- --mock --month=2026-02 --output=./reports

# テスト実行
npm test
```

## 出力レポート

### A. 経営サマリー
売上・利益・現預金の要約と前月比、総合評価

### B. 月次会計レポート
PL各項目の前月比較、主要費用科目トップ5、変動が大きい費用の検出

### C. 財務健全性レポート
流動比率、自己資本比率、現預金月商倍率、ROA等の財務指標

### D. 経営判断コメント
利益・資金繰り・固定費・売上依存・銀行評価の5観点で評価とコメントを生成

### E. リスクアラート
前月比30%以上の費用増減、売上急減、現預金減少、赤字転落等を自動検知

## ディレクトリ構成

```
src/
├── cli/                  # CLIエントリポイント
├── clients/              # freee API クライアント（認証・API呼び出し）
├── commentary/           # 経営コメント生成（テンプレート / LLM）
├── config/               # 設定・しきい値
├── domain/
│   ├── accounting/       # 会計データ整形（PL/BS解析、月次比較、異常検知）
│   ├── banking/          # 銀行評価指標
│   ├── cashflow/         # 資金繰り分析
│   └── finance/          # 財務指標計算
├── evaluators/           # 状態評価（利益/資金繰り/固定費/売上/銀行）
├── reports/              # レポート生成（JSON / Markdown）
├── services/             # freee データ取得サービス
├── types/                # 型定義
└── utils/                # ユーティリティ（フォーマット、ロガー）

tests/
├── fixtures/             # モックデータ
└── unit/                 # ユニットテスト
```

## 各ドメインの責務

| ドメイン | 責務 |
|----------|------|
| **accounting** | freee生データの整形、PL/BS解析、勘定科目分類、月次比較、異常検知 |
| **finance** | 収益性・安全性の財務指標計算（利益率、流動比率、自己資本比率等） |
| **cashflow** | 現預金推移、資金繰り余力、資金ショートリスク判定 |
| **banking** | 金融機関向け評価指標（債務償還年数、月商倍率等）、簡易スコアリング |
| **evaluators** | 数値→状態判定（優良/良好/普通/注意/危険） |
| **commentary** | 経営者向け自然言語コメント生成（LLM差し替え可能） |

## 経営判断コメントの観点

1. **利益** - 経常利益率に基づく収益性評価
2. **資金繰り** - 現預金の余力月数とリスク判定
3. **固定費負担** - 販管費率に基づく費用構造評価
4. **売上依存** - 売上の安定性・前月比推移
5. **銀行評価** - 金融機関目線での財務体質評価
